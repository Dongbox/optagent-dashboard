## Design Context

### Users

OptAgent maintainers and benchmark contributors use this dashboard while reviewing solver changes, strategy changes, and benchmark result updates. They need to answer operational questions quickly: which strategy is strongest, whether a commit caused a regression, and what happened in a specific run.

### Brand Personality

Quiet, precise, and evidence-driven. The interface should feel like an engineering instrument rather than a marketing site.

### Aesthetic Direction

Use a restrained light interface with dense tables, compact charts, and deliberate color only for status, strategies, and change detection. Avoid hero sections, decorative cards, dark neon dashboards, generic gradients, and oversized marketing typography.

### Design Principles

- Start with the usable dashboard, not an introduction.
- Make comparisons scannable before making them decorative.
- Keep data provenance visible: commits, run ids, paths, and status should be easy to inspect.
- Use charts to answer specific benchmark questions, not as ornament.
- Keep the layout stable and compact across desktop and mobile.
